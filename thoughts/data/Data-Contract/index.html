# In Nutshell

It’s an API-like formal agreement between the Data Product and the data product consumer. It specifies the guarantees about a provided data set and expectations concerning Data Product usage. The goal is to generate a well-modeled, high-quality, trusted data.

# Specification

# Promises

Info

It declares the intent of the port. Promises are not guarantee of the outcome but the Data Product will behave accordingly to them to realize its intent.

The more a Data Product keeps its promises over time the more trustworthy it is.

We find:

  • Service-level objectives: such as latency, availability
  • Deprecation policy

# Expectations

Info

It declares how it wants the port to be used. They are a way to explicitly state what promises the data product would like consumers to make regarding how they will use the port

We find:

  • Data Product consume, including team, and responsible contact.
  • Semantic model: the meaning
  • Syntax model: the structure of the exchanged data
  • Purpose of data usage
  • Terms, such as query intervals and data processing volumes
  • Start data
  • End data
  • Notice period, to cancel the contract

# Contracts

Info

Â A contract is an explicit agreement between the data product and its consumers. It is used to group all the promises and expectations that if not respected can generate penalties like monetary sanctions or interruption of service

It is the format used to state of the agreement, including:

  • Data Product provider, including team, owner, and the output ports to access.
  • Level of data access required
  • Service-level agreement
  • Costs

Important

Data contracts are the link between Data Products. They refer to a specific version of an output port. Become active when a consumer requests data access and the provider accepts the request.

Data contracts can be canceled by providers or consumers with a defined notice period (e.g when the cost become too high, or when the provider needs to implement a breaking change).

Data contracts provide stability, trust, and quality within the Data Mesh. They can be used to visualize data flow across Data Products.

# One step further

Data contracts can also be used for automation: As soon as a data contract has been concluded, permissions for the output port can be set up automatically in the Self-Serve Data Platform on an event basis. When the contract is terminated, the permissions are automatically deleted again.
